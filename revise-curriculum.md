# 6. การปรับปรุงหลักสูตร

ใช้เมื่อต้องการปรับปรุงหลักสูตรที่มีอยู่แล้วอย่างมีนัยสำคัญ เช่น เปลี่ยนโครงสร้างหลักสูตร จำนวนหน่วยกิต หรือ PLO กระบวนการแบ่งเป็น 4 ขั้นตอน คล้ายการเพิ่มหลักสูตรใหม่ แต่เริ่มจากฐานข้อมูลเดิม

{% code overflow="wrap" %}
```html
[ขั้น 1: ปรับปรุงหลักสูตร] → [ขั้น 2: ข้อมูลปรับปรุงหลักสูตร] → [ขั้น 3: รายละเอียดหลักสูตร] → [ขั้น 4: การอนุมัติหลักสูตร]
```
{% endcode %}

## แนวคิด "หลักสูตรแม่–ลูก"

การปรับปรุงไม่ใช่การแก้ทับของเดิม แต่เป็นการสร้าง "รุ่นลูก" (หลักสูตรปรับปรุง) ที่อ้างอิงกลับไปยัง "รุ่นแม่" (หลักสูตรเดิม) เมื่อรุ่นลูกได้รับอนุมัติถึงขั้นที่กำหนด รุ่นแม่จะกลายเป็น "หลักสูตรเดิม" และรุ่นลูกกลายเป็น "หลักสูตรปีหน้าปกปัจจุบัน" โดยอัตโนมัติ วิธีนี้ทำให้ระบบเก็บประวัติทุกรุ่นไว้ครบ ตรวจย้อนหลังได้

## ขั้นตอนการดำเนินงาน

**ขั้น 1 — เลือกหลักสูตรที่จะปรับปรุง**

1. ไปที่เมนู จัดการหลักสูตร → ปรับปรุงหลักสูตร
2. ค้นหาหลักสูตรที่ต้องการปรับปรุง โดยพิมพ์ในช่องค้นหา (รหัส CISA / ชื่อหลักสูตร / กรองตามคณะ)
3. ตรวจสอบข้อมูลพื้นฐานหลักสูตร
4. ระบบจะแสดง รูปแบบการปรับปรุง ให้เลือก

<figure><img src=".gitbook/assets/image (6).png" alt=""><figcaption></figcaption></figure>

| รูปแบบ                     | เมื่อไหร่ควรใช้                                                                  |
| -------------------------- | -------------------------------------------------------------------------------- |
| ปรับปรุงแบบปกติ (1 ต่อ 1)  | ปรับปรุงต่อเนื่องจากหลักสูตรเดิมหลักสูตรเดียว ได้ผลเป็นหลักสูตรใหม่หลักสูตรเดียว |
| ปรับปรุงแบบรวม (หลายต่อ 1) | นำหลักสูตรหลายหลักสูตรมารวมเป็นหลักสูตรเดียว                                     |
| ปรับปรุงแบบแยก (1 ต่อหลาย) | แยกหลักสูตรเดิมหนึ่งหลักสูตรออกเป็นหลายหลักสูตร                                  |

> ⚠️ เลือกรูปแบบให้ตรง เพราะรูปแบบนี้กำหนดว่าหลักสูตรแม่ตัวใดบ้างจะถูกเปลี่ยนเป็น "หลักสูตรเดิม" เมื่ออนุมัติผ่านขั้นตอนสภามหาวิทยาลัยอนุมัติ

**ขั้น 2–3 — กรอกข้อมูลการปรับปรุง**

1. ระบบจะ คัดลอกข้อมูลเดิม (Extract) จากหลักสูตรแม่มาตั้งต้นให้ ผู้ใช้แก้เฉพาะส่วนที่เปลี่ยน ไม่ต้องกรอกใหม่ทั้งหมด
2. โครงสร้างการกรอกเหมือนการเพิ่มหลักสูตรใหม่ (8 หมวดหมู่) ทุกประการ
3. ตรวจสอบให้ดีว่าข้อมูลที่ยกมาจากของเดิมยัง "ใช้งานในปัจจุบัน" หรือไม่ (เช่น รายชื่ออาจารย์ ค่าธรรมเนียม)&#x20;

{% hint style="warning" icon="lightbulb" %}
**เคล็ดลับ:** หลังระบบดึงข้อมูลเดิมมาแล้ว ให้ไล่ตรวจทีละหมวดตามสถานะวงกลม โดยเน้นแก้ส่วนที่เป็นเหตุผลของการปรับปรุงก่อน แล้วค่อยตรวจส่วนที่ยกมาทั้งดุ้น
{% endhint %}

**ขั้น 4 — การอนุมัติ**

ส่งกระบวนการอนุมัติแบบเดียวกับหลักสูตรใหม่ (8 ลำดับการอนุมัติ, 7 แท็บ) เมื่อผ่านลำดับการอนุมัติที่ 5 (สภามหาวิทยาลัย) หลักสูตรลูกจะกลายเป็น "หลักสูตรปีหน้าปกปัจจุบัน" และหลักสูตรแม่กลายเป็น "หลักสูตรเดิม" อัตโนมัติ
